# 🗺 Roadmap

# Phase 1: Foundation & UI Shell (Каркас)
- [X] Базовый каркас приложения (Tauri v2 + React)
- [X] Настройка ограничений окна (min-width/min-height, запрет на излом интерфейса)
- [X] Уменьшение размера билда (оптимизация зависимостей)
- [X] Кастомный Titlebar (кнопки свернуть/закрыть,原生 drag region)
- [X] Design System (SCSS, :root переменные, холодная тема, State-driven цвета)
- [X] Инициализация SQLite (настройка WAL режима, Foreign Keys)
- [X] Архитектура БД (Таблицы: games, localizations, install_states)
- [X] Интеграция Mutex<Connection> в Tauri State (потокобезопасный доступ к БД из UI)
- [X] Компонентная архитектура (Wizard-флоу: Список игр -> Детали игры -> Выбор перевода)
- [X] Взаимодействие с ОС (Интеграция rfd для выбора папки с игрой)
- [ ] Создание полного дизайна в Figma (пока опираемся на базовую сетку)

# Phase 2: Core Engine & Security (Ядро)
- [X] Базовый движок распаковки (Интеграция zip кейта, распаковка по правилам)
- [X] Система маппинга путей (install_instructions JSON: срезание лишних папок из архивов)
- [X] Неявный Safe Extractor (файлы, не описанные в JSON-манифесте, игнорируются)
- [ ] Система бэкапов (СЛЕДУЮЩИЙ ШАГ): Создание .zip архива оригинальных файлов игры перед распаковкой перевода
- [ ] Rollback (Откат): Кнопка "Удалить перевод" с развертыванием бэкапа поверх измененных файлов
- [ ] Smart File Validator (Белые списки): Жесткая блокировка .exe, .bat на уровне кода. Разрешение .dll только из dll_whitelist в манифесте.
- [ ] Hash Verification: Проверка SHA-256 скачанного архива ДО распаковки (защита от подмены файла на сервере)
- [ ] Защита от Zip-Slip: Проверка, чтобы пути из архива не выходили за пределы папки с игрой (например, ../../../Windows/System32)

# Phase 3: Network & Catalog (Сеть)
- [X] Формат Манифеста (Строгие serde структуры для JSON каталога)
- [X] Синхронизация БД при старте (Проверка пустоты БД -> Парсинг -> ON CONFLICT DO UPDATE без потери локальных путей)
- [X] HTTP-клиент (Интеграция reqwest с асинхронным стримингом)
- [X] Failover Download Manager (Автоматическое переключение primary_url -> backup_url при ошибке 403/404)
- [X] State-driven UI загрузок (Чтение чанков -> app.emit -> Слушатель listen в React -> Прогресс-бар)
- [ ] Управление временными файлами (Скачивание в %TEMP% через tempfile, автоматическое удаление .zip после распаковки)

# Phase 4: UX & Error Handling (Опыт использования)
- [ ] Локализация системных ошибок (Перевод "No rights to create folder" -> "Нет прав на запись. Запустите лаунчер от имени Администратора")
- [ ] Проверка процессов (Блокировка кнопки "Установить", если .exe игры сейчас запущен, чтобы избежать ошибки "File is used by another process")
- [ ] Кнопка "Ссылка не работает" (Отправка репорта на сервер для автоматической пометки локализации)
- [ ] Обработка нехватки места на диске (Предупреждение до начала скачивания: file_size_mb + размер бэкапа > свободного места)
- [ ] Настройки (Paths Scanner, возможность ручного сброса пути к игре)

# Phase 5: Production & Distribution (Релиз)
- [ ] Логирование (Запись всех действий лаунчера в скрытый файл для техподдержки)
- [ ] Tauri Updater (Автообновление самого лаунчера при релизах)
- [ ] Сборка инсталлятора (NSIS/WiX, добавление иконки, запись в "Установка и удаление программ")
- [ ] Code Signing (Покупка сертификата для отключения окна SmartScreen "Система Windows защитила ваш ПК")
- [ ] Модерация (Веб-форма для локализаторов: загрузка архива -> генерация install_instructions -> публикация в каталог)

# ⚠️ Известные ограничения (Технический долг)
- Форматы архивов: Поддерживается только .zip. Поддержка .7z и .rar заблокирована из-за нестабильных C-зависимостей в Rust. Требование к авторам: пересобирать в zip.
- Автопоиск: Пока заглушка. Реализуется через парсинг registry.vdf Steam и реестра Epic Games.
